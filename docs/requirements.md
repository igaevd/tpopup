# General Description

A native macOS application designed for on-the-fly text translation using OpenAI's API. The application operates in two modes: launching the app normally opens a SwiftUI-based Settings window to configure the app. Launching the application executable with a `-translate` command-line flag triggers the translation flow. The translation flow instantly displays a frameless, non-editable, minimalistic popup window at the current mouse cursor location. It automatically copies the currently selected text, sends a single JSON-formatted request to the OpenAI API to determine the language direction and translate the text, and displays the result. Both the source and translated text can be played aloud using macOS's native local Text-to-Speech (TTS). The popup dismisses automatically upon losing focus or pressing the Escape key. The technical requirements are described below in plain narrative language. Please follow best practices for implementing modern desktop macOS applications when implementing them. In other words, use modern controls, modern approaches to saving settings, and so on. I intentionally do not specify many technical details. I leave that to your discretion. If you see that there aren’t enough details, look up how and what to implement, find it, and decide on your own. Don’t overload me with technical details, endpoint names, APIs, and the like unless absolutely necessary. I wrote what I want to see from this app, and I want to get a result. Be results-oriented. And keep the progress report brief. I don’t need you to describe every function and every action you took. That’s unnecessary.

## 4. Error handling.
The point is that when starting a translation, we may have a bunch of errors, including not having a key, not having settings, and so on. Accordingly, every time we launch the app in translation mode, we need to run an absolutely minimal settings check, such as whether the key is filled in, whether we have the correct model, whether we have the EIA-Prompt, and whether there is actually text in the clipboard, because otherwise there may simply be nothing to translate. And we need to verify that it is text, because there could be, for example, some image or something else copied there. If any of this fails, then instead of the popup window that shows the translation, a regular error message appears. Just centered on the screen, using some standard macOS window that says “Error” and recommended actions. Please do not output all errors. Think of it as simply running the checks one by one. If a check fails, we immediately show an error message and do nothing else. We do not start any translations. For example, we check that the settings are filled in. We check that there is a key, a model, and an EIA-Prompt. Then we think that if all 3 are missing, we just say to fill in the settings, without even needing to provide details. If something specific is missing, for example the key is filled in but the EIA-Prompt is not, then we say to fill in the EIA-Prompt in the settings. If several items are missing, think about how to handle that elegantly. This should be very fast. We do not need to run all checks. We need to run some basic checks, and if something does not work, the window should appear immediately. Next, we check that there is text in the clipboard. We can say there is no text. So first we check the settings, then we check that there is something to translate. Only after that do we start the translation. The window and then the popup appears. Ideally, the popup should appear immediately after the checks. And before we even get the result from the server. So there is no situation where there is simply nothing on the screen. The program seems to be doing something, but we are stuck. In other words, either an error message or the window itself should appear instantly, even if it is empty and has no text. And some nice spinning icon that shows that something is being processed. As soon as processing is complete, the icon disappears. I leave the development of this design to your discretion. It should be very beautiful, but at the same time very simple and minimalist.
