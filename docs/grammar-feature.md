 A new mode needs to be added to this app called Grammar. This is for grammar correction. For this mode, a new tab needs to be added to the settings window.  When we launch the app in grammar correction mode, we simply pass the `-grammar` input parameter. We need to handle this incoming parameter similarly to how we handle the `-translate` parameter for the translation mode. We also need to handle errors in a similar way, for example, checking that all required parameters are filled in and returning those errors. In other words, the overall app structure, error handling, launch flow, and so on, as well as incoming parameter handling, should remain the same.

The `-grammar` parameter should work as grammar correction with replacement of the selected text. In order to implement it correctly, you need to understand how it will be used in practice. It will look like this. In some text editor, text input window, or other text area where I can edit text, meaning it will always be a place where I can edit text, I select the text and press a keyboard shortcut that copies this text and calls the tpopup app with the `-grammar` parameter. Copying the text is not part of this app right now, it will be handled by a Quick Action script.

But when our app starts, it gets the text from the clipboard, sends it for correction to OpenAI using the AI prompt from the Grammar settings (key, model, and prompt). Then it should insert the grammatically corrected text into the place where we selected the text. For this, I think we need to use some keyboard shortcuts, such as Command plus V.
